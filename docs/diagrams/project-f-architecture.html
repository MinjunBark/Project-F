⚠️ Known Issue — Long Run Time (20+ min observed) Root cause: Gemini API rate limiting enforces a 4s sleep between each API call per key. With 1 key and 100 leads this is ~12–15 min minimum; quota exhaustion or API errors (e.g. model deprecation mid-run) can push it to 20+ min.
Current mitigation: Dual API key parallel batching splits leads 50/50 across 2 Gemini keys, halving runtime to ~6–10 min.
To investigate further: Check Gemini quota usage in Google AI Studio · reduce sleep from 4s to 2s if RPM allows · add a 3rd API key for 3-way split.
